Git / GitHub

Lagt ut 01.10.2018

Git er et versjonskontrollsystem. Et versjonskontrollsystem er et programvareverktøy, som registrerer endringer i en fil eller et sett med filer over tid, og som husker bestemte versjoner av koden til senere. Et slikt system lar deg bl.a. tilbakestille kodefiler til tidligere tilstand, sammenligne endringer i koden over et bestemt tidsforløp, og finne ut hvem som sist endret koden, som evt. førte til et problem, og når dette skjedde.

Ved å benytte Git er det enklere å jobbe sammen på et prosjekt. Man kan slå sammen kode som man jobber på til et prosjekt, og det er mulighet til å jobbe på forskjellige steder/flere maskiner. Ved å lage en forgrening av prosjektet, kan man teste ut nye ting uten at dette får påvirkning for andre utviklere som jobber på det samme prosjektet. Git passer på at man alltid har en backup, og det kan finne feil som man har introdusert i utviklingsarbeidet.

Github er en nettbasert tjeneste for prosjekter som bruker Git.

Mine tanker

Foreløpig har jeg lite erfaring med Git og GitHub. Det krever at man bruker litt tid på å sette seg inn i GitHub, for så å lære det noenlunde ordentlig. Når man først forstår programmet, tror jeg GitHub er en superløsning, som gjør kodehverdagen enklere.
